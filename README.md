## 🌟 Características Clave

### 1. AI RAG Chat Region Plugin (`plugin_plsql_chat_ai.sql`)
* **Diseño Glassmorphism Premium:** Ventana de chat moderna con efectos traslúcidos (`backdrop-filter`), animaciones sutiles del bot y un indicador interactivo de escritura (*typing indicator*).
* **Adaptabilidad (Universal Theme):** Integra variables CSS globales de APEX para sincronizarse automáticamente con el esquema de colores de la aplicación.
* **Soporte de Ícono de Marca Personalizada:** Permite un ícono flotante personalizado (`widget_icon_url`), eliminando automáticamente bordes y círculos de fondo predeterminados en todos los estados físicos de interacción (`hover`, `focus`, `active`).
* **Memoria de Sesión Inteligente:** Utiliza `APEX_COLLECTION` para persistir contextualmente hasta $N$ chats de historial en la misma sesión activa.
* **Filtro de Categorías Dinámico:** Permite asociar un elemento de página (Page Item) para segmentar el conocimiento RAG en tiempo real de forma dinámica.

### 2. AI RAG Data Loader Process Plugin (`loader_plugin_plsql_code.sql`)
* **APEX 26.1 Callback Procedure:** Diseñado conforme a la especificación moderna de plugins de tipo proceso (`IN OUT NOCOPY` parameter signature) para evitar fallos de compilación `PLS-00306`.
* **Doble Modo de Escritura:**
  * **`REPLACE`:** Borra en una transacción segura los chunks y embeddings anteriores de un título y genera la nueva estructura desde cero.
  * **`APPEND`:** Calcula la secuencia previa del documento de forma autónoma y continúa la numeración automática de los fragmentos (ej: `[Part X+1]`).
* **Respuesta Dinámica y Auditada:** Genera un mensaje de éxito rico e interactivo en pantalla que resume las tablas creadas, el método de inserción y las dimensiones vectoriales generadas.

## 📋 Requisitos Previos y Esquema de Base de Datos

Antes de registrar los componentes en Oracle APEX, debes disponer de soporte vectorial en tu Base de Datos (Oracle 23ai / 23c).

### 1. Tabla de Conocimiento RAG (Vectores)
Crea la tabla que almacenará los fragmentos y los embeddings de 3072 dimensiones generados por Gemini:

```sql
CREATE TABLE ai_rag_knowledge (
    id          NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title       VARCHAR2(500) NOT NULL,
    content     CLOB NOT NULL,
    category    VARCHAR2(255),
    embedding   VECTOR(3072, FLOAT32),
    created_at  DATE DEFAULT SYSDATE
);
```

### 2. Tabla de Historial y Auditoría del Chat (Flotante)
Para registrar los logs de interacción con fines estadísticos y de auditoría:

```sql
CREATE TABLE ai_chat_historial (
    id             NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    usuario        VARCHAR2(255),
    pregunta       VARCHAR2(32767),
    respuesta      CLOB,
    fecha_registro DATE DEFAULT SYSDATE
);
```

## 🚀 Guía de Instalación en Oracle APEX

### Paso 1: Compilar la lógica del Cargador RAG
1. Ve a **SQL Workshop > SQL Commands** o ingresa mediante **SQLcl**.
2. Compila el código del archivo [loader_plugin_plsql_code.sql](file:///c:/Users/cristian/Desktop/plugin%20chat/loader_plugin_plsql_code.sql) en el esquema de tu aplicación APEX.

### Paso 2: Configurar los Plugins en APEX Shared Components
Para cada uno de los dos plugins de la suite:

#### A. Registro del Plugin de Región: **AI Chat RAG Region**
1. Ve a **Shared Components > Plugins > Create**.
2. **Name:** `AI Chat RAG Region`
3. **Internal Name:** `COM.CRISTIAN.CHAT.AI`
4. **Type:** `Region`
5. **PL/SQL Code:** Pega el contenido íntegro del archivo [plugin_plsql_chat_ai.sql](file:///c:/Users/cristian/Desktop/plugin%20chat/plugin_plsql_chat_ai.sql).
6. **Render Function Name:** `e_render_chat`
7. **AJAX Function Name:** `e_ajax_chat`
8. **Atributos de Región:** Registra las propiedades requeridas en Shared Components (como *API Key*, *RAG Table*, *Embedding Column*, *Welcome Message*, etc.).

#### B. Registro del Plugin de Proceso: **AI RAG Data Loader**
1. Ve a **Shared Components > Plugins > Create**.
2. **Name:** `AI RAG Data Loader`
3. **Internal Name:** `COM.CRISTIAN.LOADER.AI`
4. **Type:** `Process`
5. **PL/SQL Code:** Llama al procedimiento compilado en el Paso 1 (`e_execute_loader`).
6. **Execution Function Name:** `e_execute_loader`
7. **Atributos de Proceso:** Registra los campos obligatorios del cargador (*API Key*, *Target Table*, *Title Item*, *Content Item*, *Write Mode*, *Chunk Size*, *Chunk Overlap*).

## 💡 Ejemplos de Configuración y Uso

### ⚙️ Ejemplo 1: Flujo del Cargador RAG en Página de Carga
Para cargar datos estructurados desde una interfaz web a tu modelo de embeddings:

1. Crea un **File Browse** o un **Text Area** para ingresar tu contenido (`P10_CONTENT`).
2. Agrega un Page Item para el título del documento (`P10_TITLE`).
3. Agrega un Page Item para la categoría opcional (`P10_CATEGORY`).
4. En el flujo de procesamiento de la página, crea un nuevo **Process** asociado al plugin **`AI RAG Data Loader`**.
5. Asigna los atributos:
   * **API Key:** `AIzaSyD...` (Tu Gemini API Key)
   * **Target Table:** `AI_RAG_KNOWLEDGE`
   * **Title Item:** `P10_TITLE`
   * **Content Item:** `P10_CONTENT`
   * **Category Item:** `P10_CATEGORY` (Opcional)
   * **Write Mode:** `REPLACE` (o `APPEND`)
   * **Chunk Size:** `1000` (Caracteres aproximados por fragmento)
   * **Chunk Overlap:** `200` (Caracteres superpuestos para conservar contexto semántico)

Al ejecutar la página e ingresar un texto, el proceso fragmentará el texto, llamará asíncronamente a los modelos de embeddings de Google Gemini, y los insertará listos para la búsqueda matemática del chat.

Para renderizar el chat de consulta:

2. Agrega una nueva región a tu árbol de componentes y selecciona el tipo **`AI Chat RAG Region [Plug-In]`**.
3. En el panel de atributos de la región, configura el origen semántico:

   * **RAG Table:** `AI_RAG_KNOWLEDGE`
   * **Col ID:** `ID`
   * **Col Embedding:** `EMBEDDING`
   * **Col Content:** `CONTENT`
   * **Col Title:** `TITLE`
   * **Col Category:** `CATEGORY`
   * **Bot Name:** `Asistente Inteligente`
   * **Welcome Message:** `¡Hola! Soy tu asistente de conocimiento integrado. ¿En qué puedo guiarte hoy?`
   * **Primary Color:** `#0066cc` (Usa colores armónicos o HSL del Universal Theme)
   * **Widget Icon URL:** `#APP_FILES#tu-icono-robot-transparente.png` (Opcional para marca blanca premium)

## 🛡️ Seguridad y Blindaje SQL
Tanto el chat como el procesador del cargador dinámico aplican validaciones estrictas y sanitización total para prevenir inyecciones SQL u otros incidentes de seguridad en el backend:
* **Sanitización Dinámica:** Implementa llamadas a `DBMS_ASSERT.ENQUOTE_NAME` en los nombres de tablas y columnas variables ingresadas en Page Designer.
* **Escapado HTML Seguro:** El script del cliente implementa llamadas a `apex.util.escapeHTML(...)` contra todos los inputs de usuario antes de renderizar burbujas en el DOM, anulando vectores de ataque Cross-Site Scripting (XSS).
* **Parámetros Bindados:** Las consultas vectoriales se realizan mediante cursores referenciados seguros aplicando Binds (`USING`) para las variables vectoriales y de categorías dinámicas.
